FUNCTION
eTemplate()
const  etm = new  eTemplate ({
syncClass :  default: "et_sync" ,
openDelimiter :  default: "<%" | "two characters" , 
closeDelimiter :  default: "%>" | "two characters"
});
What it does:  function to declare

Parameters:  
• All the parameters are optional

•  syncClass : string
syncClass :  "[et_sync(default)]" 
• class name to specify elements to re-render when sync() is executed.
• If you omit this, the class name "et_sync" will be added to parent elements of templates or template blocks
• If there is no parent element, just before the template, <span> will be added as a parent element.
• If you don't want this <span>, surround templates with a tag that you want, such as <div>.
• In case a template is used in the attribute of tag, the class name will be added to the tag, itself.
(<Input class="et_sync" type="text" value="<%= data%> data-sync="data">)

•  openDelimiter : string
openDelimiter :  "[<% (default)]" 
• should be two characters that can be easily distinguished from HTML or CSS
• open delimiter for comment will be set as "openDelimiter"+"%".

•  closeDelimiter : string
closeDelimiter :  "[%> (default)]" 
• should be two characters that can be easily distinguished from HTML or CSS
render() and parameters
const  etm = new  eTemplate () ;

etm . render ({
syncUrl :  "path" , 
startUrl :  "path" , 
scrollObj : {  id :  "id" ,  block :  "start | center | end"  },
iScope :  default: '' | "body"
});
What it does:  Render html files including templates.

Parameters:  
• All the parameters are optional

•  syncUrl : string
syncUrl
:  
'/feature.html'
• path of html file to replace current html with (optional)
• Use this parameter if all the html files are rendered in single page.
• If you omit this, render() find and render startUrl. In case there is no startUrl either, current html file will be rendered.

•  startUrl : string
startUrl
:  
'/about.html'
• path of html file to render firstly replacing index.html with (optional)
• Use this parameter if no syncUrl is specified and want it to replace index.html.
• If you omit this, render() will regard index.html as the first html file to be loaded.

•  scrollObj : object
scrollObj : {  id :  "id" ,  block :  "start | center | end"  }
• id: id of element to scroll to inside the html file of the second argument, sync_url
• block: vertical alignment of the element. One of "start", "center", or "end"

•  iScope : string
iScope :  "body" 
• render() finds out linked CSS files and check templates inside them.
  Even though there is no template, all CSS elements have to be checked for templates, and it takes a while.
• If there is no template in CSS files, you can reduce a delay for checking them by setting this parameter.
• If iscope is set to "body", render() skips checking CSS files.

sync()
sync ( "body" )
What it does:  refresh only the HTML and CSS elements that has templates on current page
sync ( )
• To refresh templates in HTML and CSS when variables change from click, mouseover, and other events, call this function in the event-handling codes.
• Refreshes only the templates that are affected by changed variables including control templates like for, if, while, and so on.
• Of course, sync() should be executed after variables change

Parameter:  
•  "body" : string

• If you omit this, sync() will refresh all the templates of HTML and CSS.
• In case there is no template in CSS, you can speed up refereshing templates.
